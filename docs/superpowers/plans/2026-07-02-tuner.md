# Tuner Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the controlled, presentational chromatic-tuner display (`src/components/Tuner/`) per `docs/superpowers/specs/2026-07-02-tuner-design.md`.

**Architecture:** One pure-render component. A recessed `--stage` "screen" (arrows + big note, cents/Hz readout, −50…+50 scale with sliding needle tick, A=ref caption) over a control row that composes kit `Toggle` (mute), `NumberField` (reference Hz), `SegmentedControl` (CENT/HZ, only when `onModeChange` given). All state via `data-*` attributes; no timers, no audio.

**Tech Stack:** React + TypeScript, CSS Modules (tokens only), Vitest + Testing Library (`fireEvent`), gallery auto-registration via `import.meta.glob`.

## Global Constraints

- Tokens only — no hardcoded colors. In-tune = `--led-green*`, off-tune = `--led-orange*`, never red.
- `data-*` attributes for state; `:focus-visible` only; sizes `sm`/`md` (default `md`).
- Tests use `fireEvent`, NOT `userEvent`.
- Gate: `npx tsc --noEmit` + `npx vitest run` + `npm run build` green.
- Reduced motion: needle/LED transitions snap (`transition: none`); color/position still change.
- Fonts: `--font-display` (note letter), `--font-mono` (readouts/labels).
